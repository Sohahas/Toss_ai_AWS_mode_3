# 사용자 설치 가이드

이 문서는 컴퓨터에 익숙하지 않은 사용자가 전체 구조를 이해하고 설치할 수 있도록 만든 안내서입니다.

## 한 줄 요약

웹 화면은 Render에서 켜두고, 실제 토스증권 주문봇은 AWS 고정 IP 서버에서 24시간 실행합니다.

집 PC를 24시간 켜둘 필요가 없습니다.

## 준비물

1. GitHub 계정
2. Render 계정
3. AWS 계정
4. 토스증권 Open API Client ID / Secret
5. 토스증권 Open API에 등록할 AWS Static IP
6. OpenAI API Key
7. Telegram Bot Token / Chat ID

## 비용 구조

- Render Web: 무료 플랜 사용 가능
- Render PostgreSQL: 기록 보존용 유료 DB 사용 권장
- AWS Lightsail: 최소 Linux public IPv4 번들 권장
- OpenAI API: 사용량만큼 과금

AWS에는 반드시 Budgets 알림을 설정하세요.

## 설치 순서

### 1. Render 배포

Render Blueprint로 배포하면 아래 2개 리소스가 만들어져야 합니다.

```text
ai-stock-assistant-web
ai-stock-assistant-db
```

Render Worker는 사용하지 않습니다.

Render Web 환경변수는 아래처럼 둡니다.

```env
BROKER_MODE=toss
BROKER_API_ENABLED=false
LIVE_TRADING_ENABLED=true
MARKET_POLL_INTERVAL_SECONDS=60
ANALYSIS_INTERVAL_SECONDS=1800
```

### 2. AWS 주문봇 만들기

자세한 내용은 `AWS_고정IP_주문봇_가이드.md`를 따르세요.

핵심 순서:

1. AWS Lightsail Linux 서버 생성
2. Static IP 생성 후 서버에 연결
3. 토스증권 Open API 허용 IP에 AWS Static IP 등록
4. 서버에 프로젝트 업로드
5. `.env.aws.example`을 `.env`로 복사
6. `.env`에 실제 API 키 입력
7. `setup_aws_worker.sh` 실행
8. `run_aws_worker.sh` 실행 또는 systemd 등록

### 3. 대시보드 접속

Render Web 주소로 접속합니다.

한국어 로그인 화면이 뜨면:

- 아이디: `admin`
- 비밀번호: Render에 설정한 `DASHBOARD_PASSWORD`

브라우저 팝업이 아니라 별도 로그인 화면에서 입력합니다. 로그인은 최대 12시간 유지되며, 공용 기기에서는 오른쪽 위 `로그아웃`을 누르세요.

관리자와 큰형 계정을 나누려면 Render 웹 서비스의 `Environment`에 다음 값을 추가합니다.

- `DASHBOARD_DISPLAY_NAME`: 관리자 화면에 표시할 이름
- `VIEWER_USERNAME`: 큰형이 사용할 조회 전용 아이디
- `VIEWER_PASSWORD`: 큰형이 사용할 조회 전용 비밀번호
- `VIEWER_DISPLAY_NAME`: 예: `큰형`

관리자 계정은 자동매매와 행동패턴 설정을 모두 변경할 수 있습니다. 조회 전용 계정은 계좌·투자 판단·매매 및 운영 기록만 확인할 수 있으며 설정 변경은 서버에서도 차단됩니다.

미국 주식은 정규장에서만 달러 금액 주문으로 1주 미만 매수가 가능합니다. 프리·애프터·데이마켓은 정수 수량 지정가 주문만 가능하므로, 계좌 현금이 충분해도 AI가 정한 1회 주문 예산이 1주 가격보다 작으면 `주문 수량 부족`으로 보류됩니다.

### 4. 자동매매 시작

1. 대시보드에서 실제 보유 종목과 자산이 보이는지 확인합니다.
2. `AI 행동패턴`을 선택합니다.
3. `성향 저장`을 누릅니다.
4. 필요하면 `체결 후 OCO 손절·익절 보호`와 익절·손절 비율을 저장합니다.
5. `자동매매 시작`을 누릅니다.

이후 주문은 AWS 고정 IP 주문봇에서 자동 실행됩니다.
`주문·보호 현황`에서 접수, 부분 체결, 체결, 취소 및 OCO 감시 상태를 확인할 수 있습니다.

## AI 행동패턴

대시보드에서 드롭다운으로 바꿀 수 있습니다.

- 홀드: 신규 주문을 막고 관찰만 합니다.
- 보수적: 확실한 근거가 있을 때만 작게 거래합니다.
- 기본형 · 보수적 수익추구: 기본값입니다. 원금 방어를 우선하되 수익 기회를 탐색합니다.
- 공격적: 검증된 상승 기회가 있으면 더 적극적으로 거래합니다.
- 초공격형 · 최대수익 지향: 위험 한도를 가장 많이 엽니다. 일반 사용은 권장하지 않습니다.

최대수익 지향에서도 미수·신용 거래는 하지 않습니다.

## 수익 그래프

대시보드에서 아래 단위로 볼 수 있습니다.

- 일별: 한국시간 오늘 00시부터 24시까지 시간별 마지막 수익률
- 주별: 오늘 기준 전 3일, 오늘, 후 3일의 날짜별 마지막 수익률
- 월별: 프로그램이 처음 계좌 기록을 저장한 달부터 현재 달까지의 월별 마지막 수익률

기록이 없는 미래 날짜나 서버가 꺼졌던 빈 구간은 임의의 값으로 채우지 않고 그래프에서 공백으로 표시합니다.

그래프는 AWS 주문봇이 계좌를 갱신하면서 쌓는 기록을 사용합니다. 처음 실행한 직후에는 기록이 부족해서 그래프가 비어 보일 수 있습니다.

## 갱신 주기

- 계좌·시세 갱신: 1분마다
- AI 탐색·판단: 30분마다
- 웹 화면 새로고침: 60초마다

웹 화면과 실제 토스 계좌·시세 조회 모두 60초마다 갱신됩니다.

## 자주 생기는 문제

### 토스 인증 실패 403: IP address not allowed

토스증권에 등록된 허용 IP가 AWS Static IP와 다릅니다.

- AWS Lightsail Static IP를 확인합니다.
- 토스증권 Open API 허용 IP를 그 주소로 변경합니다.
- AWS 주문봇을 재시작합니다.

### 대시보드에 보유 종목이 안 보임

- AWS 주문봇이 실행 중인지 확인합니다.
- AWS `.env`의 `DATABASE_URL`이 Render DB의 External Database URL인지 확인합니다.
- Render Web과 AWS Worker가 같은 DB를 바라보는지 확인합니다.

### 수익률 그래프가 비어 있음

처음 실행 직후에는 정상입니다. 계좌 갱신 기록이 몇 번 쌓인 뒤 표시됩니다.

### OpenAI 429 오류

OpenAI API 크레딧 또는 결제 한도를 확인하세요. 이 시스템은 AI 판단을 30분마다 실행하도록 설정되어 있어, 속도와 토큰 사용량의 균형을 맞춘 상태입니다.

## 안전 주의

자동매매는 손실이 날 수 있습니다. 특히 초공격형·최대수익 지향은 위험이 큽니다. 처음에는 보수적 또는 기본형으로 충분히 관찰한 뒤 사용하는 것을 권장합니다.
