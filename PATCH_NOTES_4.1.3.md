# 4.1.3 패치노트

## 매매 성향

- 홀드·보수형을 제외한 기본형, 공격적, 최대수익형에서 뉴스·공시 URL 없이도 단타할 수 있습니다.
- 최근 6시간의 1분 시세를 저장하고 5·15·30·60분 변동률과 60분 가격 범위를 AI에 전달합니다.
- 기본형은 최대 종목/주문 비중 60%, 공격형은 85%, 최대수익형은 100%까지 허용합니다.
- 최대수익형은 확신·현금유지·일일손실·쿨다운 같은 소프트 제한을 사실상 해제했습니다.
- 예수금 초과, 보유수량 초과 매도, 미수·신용, 금지상품, 거래정지, 중복 주문 방지는 유지합니다.

## 주문 수량 부족 개선

- 미국 정규장 외에는 소수점 매수가 불가능하므로 정수 1주가 필요합니다.
- 기본·공격·최대수익형에서 예수금과 성향별 최대 비중이 1주를 허용하면 0주 계산을 1주로 자동 보정합니다.
- 1주를 살 현금 자체가 없으면 주문할 수 없으며, AI가 더 저렴한 후보를 선택하도록 실행 조건을 전달합니다.

## 화면과 종목명

- 메인 최근 투자 판단에는 현재 거래 가능한 시장의 기록만 표시합니다.
- 장이 닫힌 시장의 기록은 메인에서 숨기고 `최근 7일 전체 보기`에 보존합니다.
- 토스가 반환한 종목명을 판단 기록에 저장하므로 신규 종목도 숫자 코드로만 표시되지 않습니다.
- 기존 091170 기록은 `KODEX 은행`으로 표시합니다.

## DB/DNS 자동 복구

- AWS 시작 시 Render DB 주소를 찾지 못하면 2초부터 최대 60초 간격으로 계속 자동 재시도합니다.
- 실행 중 DB/DNS가 일시 중단돼도 작업봇을 종료하지 않고 다음 주기에 다시 연결합니다.
- 웹은 배포 시작 시 최대 6회 재시도한 뒤 실패를 명확히 보고합니다.
