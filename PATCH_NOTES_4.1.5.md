# 4.1.5 패치노트

## OCO 중복 등록 오류 수정

토스증권은 한 종목에 진행 중인 조건주문을 1개만 허용합니다. 기존에는 같은 종목을 추가 매수할 때 새 OCO를 생성해 `duplicate-conditional-order` 오류가 발생했습니다.

4.1.5부터는 다음 순서로 처리합니다.

1. 같은 종목의 기존 OCO 확인
   - DB뿐 아니라 토스 앱·이전 버전이 남긴 진행 중 조건주문도 조회
2. 최신 계좌에서 전체 보유수량과 평단가 재조회
3. 토스 공식 조건주문 수정 API 호출
4. 전체 보유수량 기준으로 익절·손절 가격 재계산
5. 새로 발급된 조건주문번호를 DB와 대시보드에 반영

수정 응답을 받지 못한 경우에는 새 OCO를 중복 생성하지 않습니다. 토스의 진행 중 조건주문 목록에서 수량·익절가·손절가를 대조하고, 갱신이 반영되지 않은 경우에만 기존 조건주문 수정 API를 다시 호출합니다.

이미 `duplicate-conditional-order`로 실패한 최근 매수 기록도 보호 완료로 간주하지 않고 다시 확인하므로, 4.1.5 배포 후 별도의 DB 삭제 작업은 필요하지 않습니다.

## 함께 포함된 4.1.4 화면 수정

- 모든 OCO·주문·운영 기록에서 종목명 우선 표시
- 주문번호와 조건주문번호를 별도 열로 분리
- 텔레그램에서도 `종목명(종목코드)` 형식 사용
